# Documentation Rules (Mintlify)

## Hard Rules

- **ALWAYS** use root-relative links without `.md`/`.mdx` in `docs/**`: `[Config](/configuration)`, NOT `[Config](configuration.md)`
- **ALWAYS** use anchors on root-relative paths: `[Hooks](/configuration#hooks)`
- **NEVER** use em dashes or apostrophes in headings — they break Mintlify anchor links
- **NEVER** edit `docs/zh-CN/**` or `docs/ja-JP/**` directly — these are generated by the i18n pipeline
- **ALWAYS** use full `https://docs.openclaw.ai/...` URLs in README.md (GitHub needs absolute URLs)
- **NEVER** include personal device names, hostnames, or real paths — use placeholders like `user@gateway-host`
- **NEVER** include real phone numbers, videos, or live config values — use obviously fake placeholders

## Doc Config

- Config: `docs/docs.json`
- i18n glossary: `docs/.i18n/glossary.zh-CN.json`
- i18n pipeline: `scripts/docs-i18n`
- Dev server: `pnpm docs:dev` (runs `mint dev`)
- Link check: `pnpm docs:check-links`
